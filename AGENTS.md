# 项目级指令

- 每次新对话开始时，必须完整读取 `项目概述.md` 和 `网站进度表.md`，了解项目定位与当前版本后再执行任务。
- 每次修改网站代码、功能、界面、素材、数据或运行方式后，必须同步更新 `网站进度表.md`：更新版本号、日期、相关模块状态和版本记录。
- 当功能设计、产品结构、核心交互、业务规则、项目定位、技术架构或目录结构发生变化时，必须同步更新 `项目概述.md`。
- 若一次修改同时影响项目设计和开发进度，必须同时更新 `项目概述.md` 与 `网站进度表.md`。
- 进度文档只保留当前有效信息和精简版本记录，不记录逐次操作过程或备份文件流水账。
- Git 提交信息必须使用中文，清晰概括本次变更的主要目的；不要使用英文、无意义或过于笼统的提交描述。
- `AGENTS.md`、`项目概述.md` 和 `网站进度表.md` 必须保留在仓库根目录并纳入 Git 跟踪，确保其他人 clone 项目后，AI 能直接读取项目级约束、项目概述和当前进度。
